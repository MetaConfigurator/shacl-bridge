# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in SHACL Bridge, please report it by emailing **girigov39@gmail.com**.

Please do **not** open a public issue for security vulnerabilities.

### What to Include

When reporting a vulnerability, please include:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested fixes (if available)

### Response Time

We will acknowledge receipt of your vulnerability report within 48 hours and provide a more detailed response within 5
business days.

## Security Best Practices

When using SHACL Bridge:

- Keep the package updated to the latest version
- Validate input files before processing
- Be cautious when processing documents from untrusted sources
- Review the generated documents before using it in production

Thank you for helping keep SHACL Bridge secure!
